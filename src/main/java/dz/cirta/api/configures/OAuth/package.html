package contains security configuration of all exposed based http end points, implementation of both authentication and authorization workflows. and facebook login product integration with cirta-social
by default, cirta-social do not serve any resource behind http protocol, every interaction should be encrypted, cirta-social redirect all http request to htpps in order to avoid men in the middle attacks.
cors is fully activated, for now there is only one application components who's allowed to interact with cirta-social over http protocol. any request coming outside mobile cirta social will be ignored.
cirta-social doesn't have such cross site request forgery vulnerability since all secured resources requires a valid access token, that access token cover user's session and delivered once a successful authentication just been established.
